## 💙 Loved (잘된 점)

1. **GlobalKey 기반 좌표 시스템 도입**  
   비율 추정(`screenH * 0.43`) → 실제 위젯 좌표 읽기로 **근본적인 아키텍처 전환** 성공. 화면 크기·레이아웃 변경에도 안정적.

2. **브라우저 서브에이전트 활용**  
   직접 게임을 플레이하면서 스크린샷 캡처 → 실제 픽셀 좌표를 분석하여 문제 확인. 이전 세션과 달리 **증거 기반** 디버깅.

3. **3건 버그 일괄 수정**  
   폭탄 fallback 멈춤 + 핸드 empty 멈춤 + 쌍피 피 카운팅 → 모두 한 세션에서 근본 원인 분석 후 수정.

## 📚 Learned (배운 점)

1. **Flutter GlobalKey 패턴**  
   `RenderBox.localToGlobal(Offset.zero)` + `size`로 위젯의 **실제 화면 좌표를 런타임에 읽기**. fallback 포함 안전 패턴:
   ```dart
   Offset? _getWidgetCenter(GlobalKey key) {
     final renderBox = key.currentContext?.findRenderObject() as RenderBox?;
     if (renderBox == null || !renderBox.hasSize) return null;
     final pos = renderBox.localToGlobal(Offset.zero);
     return Offset(pos.dx + renderBox.size.width / 2, pos.dy + renderBox.size.height / 2);
   }
   ```

2. **비율 기반 좌표의 함정**  
   `screenH * 0.43`은 **Expanded 위젯, SafeArea, 사이드 패널 유무**에 따라 실제와 크게 다를 수 있음. 특히 Column + Expanded 조합에서는 비율 예측이 거의 불가능.

3. **setState 누락 = 게임 멈춤**  
   Flutter에서 async 함수 내부의 모든 분기(early return 포함)에서 `setState`를 호출하지 않으면 **UI가 갱신되지 않아 사실상 멈춤**. 특히 `if (!mounted) return` 직전의 상태 갱신에 주의.

## 😔 Lacked (부족했던 점)

1. **처음부터 GlobalKey를 쓰지 않음**  
   비율 추정 → 스크린샷 측정 → 비율 재조정을 2~3회 반복한 뒤에야 GlobalKey 도입. 마스터의 제안이 있기 전에 **먼저 근본적 해결책을 제시**했어야 함.

2. **AP-004 (점진적 수정 루프) 부분 재발**  
   좌표 수정을 `_executePlay`만 먼저 → `_executeAiTurn` → `_executeBonusPlay` 순으로 나눠서 진행. 전체 영향 범위를 먼저 파악하고 일괄 수정했다면 더 효율적이었을 것.

3. **상대방 핸드 key 부착 누락**  
   덱/필드/핸드/획득은 처음에 key를 부착했으나, `_opponentHandKey`를 빠뜨려서 추가 수정 필요. **체크리스트 없이 진행**한 결과.
